UI Checklist for Developers

Print checklist

About the checklist

This checklist summarizes the guidelines given in U-LEAF UI Standards & Guidelines. You should refer to relevant sections of the guidelines for more detailed information on any of the checklist items given here.

This is not an uber-checklist. There are probably many items that could be added. More importantly, it should not be seen as a list of items that must be addressed on every functionality that you develop or defects you are working on. It is simply a guide that can be used:

  • to show the breadth of web standards.
  • as a handy tool for developers during the production phase of web applications.
  • as an aid for developers who are interested in moving towards web standards.

Instructions

  1. Complete the identification details in the header.
  2. Check each element and mark compliance Met , Not Met or N/A (Not Applicable) column.
  3. Add comments or notes as necessary, with constructive suggestions for change.

One of the difficulties associated with testing design elements on different platforms is that the test results can be subjective. For example, you may create a CSS-driven site that works perfectly on newer browsers, but doesn’t look that hot in older browsers such as IE8. The tester may interpret this as a failure point rather than acceptable "design degradation".

To avoid the time and effort involved in re-submitting pages for testing, spell out acceptable cross-browser, cross-platform differences in advance – including protocols for supporting old browsers.

At the end of the final prototyping stage, ask the designer to provide a series of screenshots that have been annotated to identify the differences in the ways the site displays on different browsers and platforms. These screenshots can be used in later testing to identify the displays that are deemed acceptable on each platform. Testers can then compare the protocols and screenshots with their findings.

Navigation

N Navigation Met Not Met N/A
N1 Is the page/portlet navigation easy to understand?
N2 Are navigation buttons consistently located?
N3 Main navigation links are clearly and consistently labeled?
N4 Have all push buttons on the screen been given appropriate shortcut keys?

Colours

C Colors Met Not Met N/A
C1 Is the colour used consistently?
C2 Do the fields and general screen background use the correct colours?
C3 Text colour has sufficient contrast with background colour?
C4 Displays colours as expected on different monitors?
C5 Are the screen colours suitable for colour-blind users?
C6 Colour is not used alone to convey meaning?
C7 Use of colour and graphics enhances rather than distracts from the site?

Content

CN Content Met Not Met N/A
CN1 Fonts, font sizes, and font colours are consistently used according to the guidelines specified? No more than one web font is used?
CN2 Font styles or sizes are not hard-coded?
CN3 Are the fonts easily readable, degrade gracefully and look correct on all accepted screen resolutions?
CN4 Are all text, form elements and labels properly aligned?
CN5 Are all the headings left aligned?
CN6 Are basic components (e.g., title bars, horizontal and vertical scroll bars) used consistently and according to established standards?
CN7 Is the layout arranged symmetrically with adequate spacing between components?
CN9 All field labels/names are not technical labels, but names meaningful to the users of the system?
CN10 Are all static text words spelt correctly and abbreviations used only when space is limited?
CN11 Is the content free from factual errors, typos, or grammatical errors?
CN12 Do Titles, Sub Titles, Sub Menu items, Tooltips for Grid Icons and Labels use Title style capitalization? e.g. User Management, Alert Maintenance.
CN13 Do Action Buttons use small style capitalization? e.g. reset, apply, search, mark as read only
CN14 Are command buttons all of similar size, shape and use the same font & font size?

Form & Validations

CN Form Met Not Met N/A
F1 Forms are navigable using the keyboard?
F2 Are mandatory fields marked clearly on the screen? (red asterisk)
F3 Are default values provided in fields, where appropriate?
F4 Are field sizes large enough to handle all valid entries? (maxlength)
F5 Are descriptive labels provided for all the fields?
F6 Are disabled fields avoided in the Tab sequence?
F7 Are all the read-only fields avoided in the Tab sequence?
F8 Does the TAB order specified on the screen go in sequence from Top Left to Bottom Right? And SHIFT + TAB order go in sequence backwords. This the default unless otherwise specified?
F9 Does the screen avoid situations where users can make errors by providing only appropriate choices and making potentially dangerous actions reversible or recoverable?
F10 Are meaningful, concise messages provided when an error or problem exists?
F11 Do error messages include guidance as to what constitutes a correct entry (e.g. the number must be numeric)?
F12 Do error messages follow project standards for consistent terminology (e.g. use of Nominal No., Nominal Number, or Nominal URN)?
F13 Are colours used following the project standards for consistency and according to the guidelines specified in Guidelines?

Browser Compatibility & Screen resolution

BC Browser Compatibility & Screen resolution Met Not Met N/A
BC1 Does the screens functionality perform as required in IE 11 and current versions of Mozilla Firefox and Google Chrome?
BC2 Displays suitably when window is resized?
BC3 Displays suitably when fonts are resized?
BC4 Displays without horizontal scrolling at 1024×768 and higher resolutions?
BC5 Checked on common variations of Screen Resolution and display suitably?

eGIF

Priority 1

SV Screen Validation Met Not Met N/A
EGP1 Provided equivalent alternatives to auditory and visual content?
EGP2 Ensured that all information conveyed with colour is also available without color?
EGP3 Does all lists and list items marked up properly?
EGP5 For data tables, identify row and column headers. For example, in HTML, use TD to identify data cells and TH to identify headers?
EGP6 Organize documents so they may be read without style sheets?
EGP7 Make programmatic elements such as scripts and applets directly accessible or compatible with assistive technologies if functionality is important and not presented elsewhereFollowing?
EGP8 Create a logical tab order through links, form controls, and objects?
EGP9 Provided keyboard shortcuts to important links (including those in client-side image maps), form controls, and groups of form controls?
EGP10 Used W3C technologies and guidelines?
EGP11 Divide large blocks of information into more manageable groups where natural and appropriate?
EGP12 Associate labels explicitly with their controls. For example, in HTML use LABEL and its "for" attribute?
EGP13 Clearly identify the target of each link?
EGP14 Use navigation mechanisms in a consistent manner?
EGP15 Provide navigation bars to highlight and give access to the navigation mechanism?
EGP16 Use the clearest and simplest language appropriate for a site's content?
EGP17 Create a style of presentation that is consistent across pages?

Priority 2

SV Screen Validation Met Not Met N/A
EGP2-1 Ensured that foreground and background colour combinations provide sufficient contrast when viewed by someone having colour deficits or when viewed on a black and white screen?
EGP2-2 When an appropriate markup language exists, used markup rather than images to convey information?
EGP2-3 Used style sheets to control layout and presentation?
EGP2-4 Used relative rather than absolute units in markup language attribute values and style sheet property values?
EGP2-5 Used header elements to convey document structure and use them according to specification?
EGP2-6 Does all lists and list items marked up properly?

Accessibility

AC Accessibility Met Not Met N/A
AC1 Are “alt” attributes used for all descriptive images, buttons, Links etc?
AC2 Does the site use relative units rather than absolute units for text size?
AC3 Do any aspects of the layout break if font size is increased?
AC4 Does the site use accessible forms?
AC5 Does the site use accessible tables?
AC6 Is there sufficient colour brightness/contrasts?
AC7 Is colour alone used for critical information?
AC8 Are all links and command buttons descriptive?
AC9 Is the content accessible with CSS switched off or not supported?
AC10 Is the content accessible with images switched off or not supported?
AC11 Does the site work well when printed?
AC12 Are keyboard commands provided for frequent users and those who prefer keyboard use over the mouse?
AC13 Does each element on a page is reachable and can be manipulated via the keyboard?
AC14 Are shortcut keys provided, where appropriate, as a method for activating controls on pulldown or cascading menus using the keyboard, without first having to select the menu (e.g., Ctrl+S saves the document)?
AC15 Are standards followed for mouse and keyboard navigation, manipulation, and interaction (e.g., TAB, SHIFT+TAB, double-click, selection of elements and objects, drag and drop)?
AC16 Is disabling used to indicate to the user when actions (including selection of specific menu items and use of objects such as checkboxes) are not valid, temporarily unavailable or do not pertain to the current state of the window?
AC17 Is an animation control used to indicate system activity when the total duration of the operation is unknown (e.g., animation clips of a magnifying glass rotating over a document can be used to indicate that a Find operation is underway)?
AC18 Is the focus indicator visible when a user uses the keyboard to tab from element to element and were it always be apparent where they are?
AC19 Is content structurally separate from navigational elements?
AC20 Configured frames with frame titles and placed meaningful content in the noframes area?

Ease of use (Usability)

EU Ease of use Met Not Met N/A
EU1 Are help (tooltip) facilities provided as appropriate?
EU2 Are selection options clear?
EU3 Is the terminology appropriate for intended users?
EU4 Is there minimal scrolling and resizeable screens?
EU5 Are messages understandable?
EU6 Are confirmation messages available as appropriate?
EU7 Are names in command button & option box names are not abbreviations?
EU8 Are fonts too large or too small to read?
EU9 Are all dropdowns on this screen sorted correctly? Alphabetic sorting is the default unless otherwise specified. Primary navigation is an exception.
EU10 Is all date entry required is in DD/MM/YYY format?
EU11 Do all the fields edit boxes indicate the number of characters they will hold by there length? e.g. a 30 character field should be a lot longer
EU12 Balance of text/graphics/white space on page.
